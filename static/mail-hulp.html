Mail (Gmail) werkend — stappen

Als mail niet werkt, moet Google een geldige refresh-token geven. Dat kan alleen in de browser (eenmalig), daarna kun je alles vanaf de terminal doen.

Staat er in .env nog een oude GOOGLE_GMAIL_REFRESH_TOKEN (van Vercel)? Dan wint die boven je nieuwe koppeling. Zet lokaal GOOGLE_GMAIL_USE_STORED_LINK_ONLY=true of haal die regel weg — zie .env.example.

  1. In de projectmap: npm start — open login → Gmail koppelen en rond OAuth af.
  2. Terminal: npm run mail:sync-env (zet de token uit .google_gmail_token.json in .env).
  3. Terminal: npm run mail:doctor — moet “Gmail API werkt” tonen. Zo niet: opnieuw stap 1.
  4. Productie: npm run mail:publish (Vercel-env + deploy), of npm run mail:publish-all als je stap 2 net deed.

In Google Cloud moet bij je OAuth-client de redirect exact zijn: https://reng-help-desk.vercel.app/api/auth/gmail/callback (en lokaal http://localhost:3000/api/auth/gmail/callback).
