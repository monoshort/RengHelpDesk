Gmail koppelen (Google OAuth)

Verstuur klantmail vanuit het dashboard via de Gmail API in plaats van SMTP. Zet in .env: GOOGLE_CLIENT_ID en GOOGLE_CLIENT_SECRET (Google Cloud Console → APIs & Services → Credentials → OAuth 2.0 Client).

Error 400: redirect_uri_mismatch = de callback-URL hieronder staat niet exact in Google Cloud bij dezelfde OAuth-client als je client-id. Dat heeft niets te maken met welk Gmail-adres je kiest; Google toont soms alleen je standaardaccount op de foutpagina.

Error 401: disabled_client (“The OAuth client was disabled”): in Google Cloud Console → APIs & Services → Credentials → open de OAuth 2.0 Client ID die exact bij je GOOGLE_CLIENT_ID hoort (zelfde client als in Vercel / .env). Zet die client weer aan (niet verwijderd/archief). Controleer ook dat je niet per ongeluk een oud of ander project gebruikt.

Error 403: access_denied / “support email” / “alleen testers”: de app staat op Testing of is niet geverifieerd. Voeg Test users toe (OAuth consent screen) voor elk Gmail-adres, of vul het consent-scherm volledig in en controleer het User support email-adres. Zie ook de punten hieronder.

  1. Schakel Gmail API in voor je Google Cloud-project.
  2. OAuth consent screen → Scopes: voeg gmail.send (versturen) en gmail.readonly (Postvak IN op /mail.html) toe. Zonder gmail.send: Insufficient Permission bij versturen. Zonder gmail.readonly: geen inbox op de Mail-pagina. Daarna app-toegang bij Google intrekken en hier opnieuw koppelen.

  4. OAuth consent screen: als de app op Testing staat, voeg Test users toe voor elk adres waarmee je wilt inloggen (bijv. toddiehelpdesk@gmail.com).
  5. Bij Google-inloggen: na “Inloggen…” zou je e-mail + wachtwoord en daarna het accountkiezer-scherm moeten zien (geen stille overschakeling naar een ander account). Werkt dat niet: probeer een incognito/private venster of log eerst uit bij Google (accounts.google.com/Logout) en probeer opnieuw.
  6. Afzender: GOOGLE_GMAIL_FROM in .env (moet overeenkomen met het account waarmee je OAuth doet).

Token: lokaal in .google_gmail_token.json. Met DATABASE_URL op Vercel wordt na koppelen in Postgres opgeslagen (aanbevolen). Optioneel kun je GOOGLE_GMAIL_REFRESH_TOKEN in Vercel zetten; werk die bij of haal weg na opnieuw koppelen. Bij conflicten: GOOGLE_GMAIL_FORCE_ENV=true laat het env-token leidend zijn.

Als Google tijdelijk “onveilige activiteit” of te veel inlogpogingen meldt: wacht een uur, bevestig je identiteit via g.co/verifyaccount, en koppel daarna zo min mogelijk opnieuw (één gedeelde GOOGLE_GMAIL_REFRESH_TOKEN op Vercel scheelt veel OAuth vanaf verschillende IP’s).
